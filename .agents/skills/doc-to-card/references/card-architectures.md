# Card Architectures

这个文件定义可复用的 card sequence patterns / 卡片顺序模式。选择 architecture 时，先看来源材料类型和读者要完成的动作，而不是先想标题。

## 选择规则 / Selection Rules

- 工具或产品文档优先选“工具文档型”。
- Step-by-step 教程优先选“教程型”。
- 观点、判断、趋势分析优先选“观点型”。
- Agent Skill、SOP、操作规范优先选“Skill 工作流型”。
- 用户需要判断新旧方法、工具差异或方案取舍时，选“对比型”。

## 工具文档型

结构：是什么 -> 什么时候用 -> 如何开始 -> 工作流 -> 常见坑 -> 练习任务

适合：

- AI 工具文档
- 产品更新说明
- 新功能解释
- API 或命令行功能介绍

卡片任务：

1. 是什么：用一句话定义功能，避免夸大。
2. 什么时候用：列出具体使用场景。
3. 如何开始：说明前置条件和第一步。
4. 工作流：展示从输入到输出的流程。
5. 常见坑：写清限制、误用和需要验证的信息。
6. 练习任务：给读者一个可执行小任务。

## 教程型

结构：问题 -> 心智模型 -> 步骤 -> 示例 -> 检查清单 -> 下一步

适合：

- 教学文章
- 操作教程
- 工作流 walkthrough
- 创作者经验复盘

卡片任务：

1. 问题：指出读者当前卡在哪里。
2. 心智模型：给一个理解框架。
3. 步骤：列出关键动作。
4. 示例：展示一个最小可行产物。
5. 检查清单：帮助读者自检。
6. 下一步：引导读者实践或深入。

## 观点型

结构：主张 -> 证据 -> 影响 -> 限制 -> 行动建议

适合：

- 技术观点
- 趋势判断
- 方法论文章
- 产品策略解读

卡片任务：

1. 主张：明确作者判断。
2. 证据：列出来源支撑，避免空泛判断。
3. 影响：说明对目标读者意味着什么。
4. 限制：写清不能推出什么。
5. 行动建议：给出谨慎、可执行的下一步。

## Skill 工作流型

结构：触发场景 -> 输入 -> workflow -> output contract -> quality check -> failure boundary

适合：

- Agent Skill
- SOP
- 内部培训流程
- 可复用内容工作流

卡片任务：

1. 触发场景：什么时候调用这个 Skill 或流程。
2. 输入：需要哪些材料和约束。
3. Workflow：执行顺序。
4. Output contract / 输出契约：产物应包含什么。
5. Quality check / 质量检查：如何判断产物合格。
6. Failure boundary / 失败边界：什么时候停止、追问或拒绝。

## 对比型

结构：旧方式 -> 新方式 -> 差异 -> 适用场景 -> 风险 -> 选择建议

适合：

- 新旧工具对比
- 手动流程与 Agent workflow 对比
- 两种内容生产方式对比
- 方案选型

卡片任务：

1. 旧方式：描述原流程的真实成本。
2. 新方式：描述新流程，不夸大。
3. 差异：用表格或要点列出改变。
4. 适用场景：说明谁适合。
5. 风险：列出限制和待验证点。
6. 选择建议：给出条件化建议。

## 调整规则 / Adaptation Rules

- 卡片数少于 architecture 步骤时，合并相邻任务，不删除 evidence boundary。
- 卡片数多于 architecture 步骤时，拆分复杂步骤或增加案例卡。
- 平台要求短内容时，保留 source facts 和 boundaries，压缩解释。
- 技术深读时，增加机制、限制、证据卡，不增加无来源观点。
