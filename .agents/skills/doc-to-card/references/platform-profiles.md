# Platform Profiles

这个文件定义不同平台的表达、节奏和制作注意点。平台适配不能改变 source facts / 来源事实。

## 小红书

- 适合目标：轻知识、工具入门、方法卡、创作者实践。
- 推荐节奏：强开头，短段落，一张卡一个动作。
- 标题策略：问题型、误区型、结论型；不要夸大结果。
- 信息密度：轻到中，每张卡 1 个主点和 2-4 个短 bullet。
- 视觉方向：大标题、步骤编号、对比小表、图标化提示。
- CTA：收藏、试一次、评论使用场景。
- 注意：避免“私藏神器”“爆款模板”“一键提效”等套路。

## 公众号

- 适合目标：解释复杂概念、沉淀方法论、连接案例和长文。
- 推荐节奏：概念递进，允许较长解释，卡片可作为 section 总结。
- 标题策略：问题型、步骤型、总结型。
- 信息密度：中，每张卡可以包含短段落和列表。
- 视觉方向：概念卡、流程卡、引用卡、section divider。
- CTA：继续阅读、保存 checklist、尝试一个案例。
- 注意：不要把小红书口吻直接搬进长文。

## LinkedIn Carousel

- 适合目标：面向国际技术读者、产品经理、开发者或创作者展示方法。
- 推荐节奏：每页一个 conclusion，逻辑递进清楚。
- 标题策略：结论型、对比型、lesson learned。
- 信息密度：中到高，但句子要短。
- 视觉方向：干净排版、流程图、对比表、少量关键词。
- CTA：comment with use case、read the source、try the workflow。
- 注意：如果用户要求中文优先，可保留英文技术词，不必强行全英文。

## 内部 wiki / 团队知识库

- 适合目标：团队执行、培训、SOP、复盘。
- 推荐节奏：先说明何时使用，再给步骤和检查点。
- 标题策略：任务型、风险提醒型、步骤型。
- 信息密度：中到高。
- 视觉方向：流程图、SOP 卡、检查清单、责任边界。
- CTA：按步骤执行、提交产物、记录失败案例。
- 注意：少用社媒语气，强调可执行和可追踪。

## 幻灯片 / 分享材料

- 适合目标：工作坊、团队分享、公开演讲、课程材料。
- 推荐节奏：一页一个观点，讲者可展开。
- 标题策略：结论型、问题型、过渡型。
- 信息密度：低到中，正文不宜太满。
- 视觉方向：大标题、结构图、案例图、对比页。
- CTA：下一页、现场练习、课后任务。
- 注意：视觉建议要考虑投影可读性，避免小字密集。
